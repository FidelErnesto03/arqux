# BLP-006: protocol.onboard + ARQUX_AGENT_ROLE + limpieza .bak

---

## §1: Planteamiento del Problema

**Tres issues de BLP-005/008 de CYCLE-04:**

1. **protocol.onboard no existe:** El workflow w06-agent-adoption.md referencia `protocol.onboard` como meta-handler, pero el handler real es `protocol.adopt`. El agente invoca un handler inexistente.

2. **ARQUX_AGENT_ROLE no configurado:** Sin esta variable de entorno, el MCP server asume rol `auditor` y bloquea todas las mutaciones (blueprint.create, complete, approve → PERMISSION_DENIED).

3. **brain.cortex.bak persiste:** Archivo de 13,937 bytes en `.arqux/brain.cortex.bak` que causa FAIL en health check.

---

## §2: Objetivo

1. Crear handler `protocol.onboard` como alias/wrapper de `protocol.adopt`
2. Configurar `ARQUX_AGENT_ROLE=governor` en el entorno MCP
3. Mover `brain.cortex.bak` → `backups/` o eliminar

---

## §6: Alcance y Exclusiones

**Dentro del alcance:** Crear protocol.onboard (1 línea alias de adopt), configurar env var, limpiar .bak.  
**Fuera del alcance:** Modificar protocol.adopt. Cambiar lógica de adopción.
