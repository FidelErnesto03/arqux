# BLP-007: Fix — blueprint handlers resuelven path sin respetar ciclo activo

---

## §1: Planteamiento del Problema

Los handlers `blueprint.read`, `blueprint.synthesize` y `blueprint.update` buscan un BLP por ID sin scoping al ciclo activo. Cuando existen múltiples ciclos con el mismo BLP ID (ej: BLP-001 en CYCLE-04 y BLP-001 en CYCLE-05), el handler siempre resuelve al primer archivo encontrado en el filesystem, ignorando el parámetro `path` o el contexto del ciclo.

**Descubierto durante esta sesión:** Al crear BLP-001 en CYCLE-05 y ejecutar `blueprint.synthesize`, el contenido se escribió en CYCLE-04/blueprints/BLP-001.md en lugar de CYCLE-05/blueprints/BLP-001.md.

**Handlers afectados:**
- `blueprint.read(bp_id)` → lee de CYCLE-04 en vez de CYCLE-05
- `blueprint.synthesize(bp_id, content, path=...)` → ignora path, escribe en CYCLE-04
- `blueprint.update(bp_id, section=N, content=..., path=...)` → ignora path, actualiza CYCLE-04

---

## §2: Objetivo

Corregir la resolución de path en los handlers de blueprint para que:
1. Respeten el parámetro `path` cuando se proporciona explícitamente
2. Si no hay path, usen el ciclo activo (vía `find_project_root` + `cycle.current`)
3. Busquen el BLP **solo** dentro del directorio del ciclo correcto, no globalmente

---

## §3: Precondiciones

- `cycle.current` handler existe y retorna el ciclo activo
- `find_project_root` existe en `arqux.state`
- Los BLPs están en `{project_root}/.arqux/cycles/{cycle_id}/blueprints/BLP-NNN.md`

---

## §6: Alcance y Exclusiones

**Dentro del alcance:**
- `blueprint.read`: scoping por ciclo activo
- `blueprint.synthesize`: respetar path explícito
- `blueprint.update`: respetar path explícito
- `_resolve_blueprint_path()` helper centralizado

**Fuera del alcance:**
- Otros handlers (task, cycle, evidence)
- Cambios en la estructura de directorios

---

## §8: Diseño Técnico

**Raíz del problema:** La función que resuelve el path del BLP busca en **todos** los directorios de ciclos:

```python
# Código actual (simplificado)
def _find_blueprint(bp_id, root):
    cycles_dir = root / ".arqux" / "cycles"
    for cycle_dir in cycles_dir.iterdir():
        bp_path = cycle_dir / "blueprints" / f"{bp_id}.md"
        if bp_path.exists():
            return bp_path  # ← retorna el PRIMERO que encuentra
```

**Fix propuesto — Helper centralizado:**

```python
def _resolve_blueprint_path(bp_id, path=None, ctx=None):
    """Resuelve el path de un BLP scoped al ciclo correcto."""
    root = find_project_root(path)
    
    # 1. Si path es explícito, buscar solo en ese directorio
    if path:
        cycles_dir = Path(path)
        if cycles_dir.name == "blueprints":
            bp_path = cycles_dir / f"{bp_id}.md"
        else:
            # Asumir que path apunta al directorio del ciclo
            bp_path = cycles_dir / "blueprints" / f"{bp_id}.md"
        if bp_path.exists():
            return bp_path
        raise FileNotFoundError(f"BLP {bp_id} not found at {path}")
    
    # 2. Usar ciclo activo
    active_cycle = _get_active_cycle_id(root)
    if active_cycle:
        bp_path = root / ".arqux" / "cycles" / active_cycle / "blueprints" / f"{bp_id}.md"
        if bp_path.exists():
            return bp_path
    
    # 3. Fallback: búsqueda global (solo lectura)
    for cycle_dir in (root / ".arqux" / "cycles").iterdir():
        bp_path = cycle_dir / "blueprints" / f"{bp_id}.md"
        if bp_path.exists():
            return bp_path
    
    raise FileNotFoundError(f"BLP {bp_id} not found")
```
