# BLP-001: Auditar y corregir el proceso de creacion de blueprints en ARQUX

---

## §1: Problem Statement

El flujo de creacion de blueprints en ARQUX tiene una falla critica: `blueprint.define` solo actualiza metadatos en el frontmatter del archivo BLP, pero **no escribe los cuerpos de las 18 secciones**. Esto produce blueprints con contenido placeholder que no son ejecutables. Adicionalmente, no existe un proceso de maduracion formal que valide cada seccion contra el Architect antes de marcar calidad.

**Evidencia:**
- BLP-001 fue creado con `blueprint_create` y `blueprint_define`; las secciones §1-§18 quedaron con texto placeholder
- `LNG:despu_s` ya documentaba: "blueprint_define solo actualiza frontmatter y metadatos — no garantiza que el cuerpo del documento este poblado"
- Los quality gates fueron marcados como ✅ sin pasar por maduracion con el Architect

**Impacto de no resolver:**
Blueprints no ejecutables, maduracion frustrante, dependencia de edicion manual, violacion del principio dogfooding (el framework no se gobierna correctamente a si mismo).

---

## §2: Objective

Auditar el proceso completo de creacion de blueprints en ARQUX, identificar las fallas en `blueprint.define` y en el flujo de maduracion, y corregirlas para que los blueprints futuros sean generados correctamente con todas las secciones pobladas.

---

## §3: Preconditions

- [ ] Ciclo CYCLE-01 existe y tiene MANIFEST.md definido
- [ ] BLP_TEMPLATE.md existe en `src/arqux/templates/`
- [ ] El handler `blueprint.define` esta implementado en `src/arqux/handlers/blueprint.py`

## §4: Guiding Principle

Dogfooding: el framework debe gobernarse a si mismo. Si el proceso de blueprints esta roto, se arregla dentro de un blueprint.

**Evidencia del problema:** Este mismo BLP-001 fue generado con secciones placeholder.

**Impacto si se viola:** El framework produce artefactos de governance inconsistentes.

## §5: Context — Flujo de gobierno

_Arqux separa la produccion de Blueprints en DOS flujos independientes, siguiendo el modelo DIALECT: creacion/maduracion (pre-ejecucion) y ejecucion/validacion/cierre (post-maduracion)._

### §5a: Flujo de Creacion y Maduracion

```puml

:BLP-NNN.md en draft;

note right

  - blueprint.create() es la unica via
  - No write_file directo

  - No ejecutar sin maduracion completa

:Indexar las 18 secciones del template.
**CRITICO:** NO escribir todo el BLP de una vez.;

partition "Fase 1 — Co-diseno (seccion por seccion)" {

  while (Quedan secciones pendientes?) is (si)

    :Seleccionar siguiente seccion
(orden §1 → §18);

    :Presentar proposito al Arquitecto;
    :Recibir especificaciones y feedback;

    while (Seccion requiere ajustes?) is (si)

      if (Requiere diagrama PUML?) then (si)
        :Disenar diagrama PUML;
        :cortex.render.diagram();
        :Arquitecto revisa y aprueba;
      else (no)
        :Refinar contenido textual;
      endif

      :Presentar seccion al Arquitecto;

      if (Arquitecto aprueba esta seccion?) then (si)
        :blueprint.update(section="§N");
        :evidence.record("seccion aprobada");
        :Pasar a siguiente seccion;
      else (no)
        :Registrar ajustes solicitados;
      endif

    endwhile (no)

  endwhile (si)

  :Todas las 18 secciones refinadas;
}

partition "Fase 2 — Quality Gate" {

  :Evaluar 6 quality gates (§18);

  if (Todos los gates cumplen?) then (si)
    :Marcar BLP como candidato;
  else (no)

      :Registrar excepcion;
    else (no)
      :Convertir en feedback pendiente;
      detach
    endif

  endif

  :Presentar BLP completo al Arquitecto;

    :Marcar BLP listo para apertura;
  else (no)
    :Registrar observaciones;
  endif

partition "Fase 3 — Apertura Controlada" {

  if (Arquitecto confirma?) then (si)

    :blueprint.ready(BLP-NNN);
    :evidence.record("BLP abierto");
    :Informar: "Listo para execute flow";

  else (no)
    :Mantener en maturing;
  endif

stop

@enduml
```

### §5b: Flujo de Ejecucion, Validacion y Cierre

```puml
@startuml
title BLP Execute Flow — Ejecucion, validacion y cierre

start

:blueprint.read(BLP-NNN);
:Verificar estado;

if (Estado = ready?) then (si)
  :blueprint.assign(executor);
  :blueprint.claim();
elseif (Estado = in_progress?) then (si)
  :Retomar progreso;
elseif (Estado = blocked?) then (si)
  :Resolver bloqueo con Arquitecto;
  if (Resuelto?) then (no)
    :Mantener blocked;
    stop
  endif
else (no)
  :Estado no ejecutable;
  stop
endif

partition "Fase 1 — Preparacion" {
  :Verificar precondiciones (§3);
  if (Cumplidas?) then (no)
    if (Arquitecto autoriza?) then (no)
      stop
    endif
  endif
  :Presentar plan operativo;
}

partition "Fase 2 — Ejecucion (tarea por tarea)" {

  repeat
    :Seleccionar siguiente tarea de §14;
    :blueprint.task(id, status="in_progress");
    :Ejecutar tarea segun §8 y §9;

    if (Bloqueo?) then (si)
      :HALT y registrar evidencia;
      if (Fatal?) then (si)
        :blueprint.cancel();
        stop
      else (no)
        if (Arquitecto aprueba?) then (no)
          :blueprint.fail();
          stop
        endif
      endif
    endif

    :Recolectar evidencia;
    if (Suficiente?) then (si)
      :blueprint.task(id, status="completed", evidence);
      :evidence.record("tarea completada");
    endif

  repeat while (Quedan tareas?) is (si)

}

partition "Fase 3 — Validacion (AC por AC)" {

  :Ejecutar validaciones tecnicas (§13);

  if (OK?) then (si)

    repeat
      :Seleccionar siguiente AC de §12;

      if (AC cumple?) then (si)
        :blueprint.ac(id, status="verified");
      else (no)
        :blueprint.ac(id, status="failed");
        :blueprint.re_delegate() — AUTOMATICO;
        note right: Max 3 intentos
        detach
      endif

    repeat while (Quedan ACs?) is (si)

    :Presentar resumen al Arquitecto;
    if (Aprueba?) then (si)
      :blueprint.approve();
    else (no)
      :blueprint.re_delegate();
    endif

  else (no)
    :blueprint.re_delegate();
  endif

}

partition "Fase 4 — Cierre" {
  if (Arquitecto confirma?) then (si)
    :Cierre via cycle.close;
  else (no)
    :Cierre diferido;
  endif
}

stop

@enduml
```
